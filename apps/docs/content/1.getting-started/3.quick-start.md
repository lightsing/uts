This guide walks you through timestamping a file using the UTS CLI and verifying the timestamp.

## Step 1: Prepare a File

Create or choose any file you want to timestamp:

```bash
# Create a sample document
echo "This document was created on $(date)" > document.txt

# Or use an existing file
ls -la document.txt
```

## Step 2: Timestamp the File

Use the `uts stamp` command to create a timestamp:

```bash
uts stamp document.txt
```

This command:

1. Computes the Keccak256 hash of `document.txt`
2. Submits the hash to the calendar server
3. Receives an initial `.ots` file with a commitment signature
4. The calendar will batch your digest with others and attest on-chain

### Output

You'll receive:

- `document.txt.ots` — The OpenTimestamps attestation file
- A commitment hash representing your timestamp request

```
File: KECCAK256 875bb58d572d099f512664e0091614a9bb96a25dc24ca849e132b5aeb0c8e952 document.txt
Internal Merkle root: b4b22388bafd543783996b70da3f67712be082374f306fef6261ee9eaa638113
Submitting to remote calendar: https://lgm1.calendar.test.timestamps.now/
Successfully wrote timestamp for document.txt
```

## Step 3: Wait for Attestation

The calendar batches digests periodically (typically every 10 seconds). Wait for the next batch to be attested on-chain.

## Step 4: Upgrade to Self-Contained Proof

The initial `.ots` file contains a pending attestation. Upgrade it to include the full on-chain proof:

```bash
uts upgrade document.txt.ots
```

This fetches the complete Merkle proof and attestation data from the calendar.

## Step 5: Verify the Timestamp

Once the batch is attested, verify your timestamp:

```bash
uts verify document.txt
```

### Verification Output

```
Digest matches: 875bb58d572d099f512664e0091614a9bb96a25dc24ca849e132b5aeb0c8e952
EAS Timestamped
        time attested: 2026-03-13T14:45:13+08:00[Asia/Hong_Kong]
        merkle root: 1148a280151da88c27d79e33dc7929074cc7f0aa0288edf1aee166ddce7552ed
```

## Using the TypeScript SDK

Programmatic timestamping with the TypeScript SDK:

```typescript
import { UTSClient } from '@uts/sdk'
import { readFileSync } from 'fs'
import { createHash } from 'crypto'

// Initialize client
const client = new UTSClient({
  calendarUrl: 'https://lgm1.calendar.test.timestamps.now'
})

// Create digest from file
const file = readFileSync('document.txt')
const hash = createHash('sha256').update(file).digest('hex')

// Submit for timestamping
const { otsFile, commitment } = await client.stamp(hash)

// Save OTS file
writeFileSync('document.txt.ots', otsFile)

console.log('Timestamped! Commitment:', commitment)

// Later: Verify timestamp
const result = await client.verify(otsFile)
console.log('Verified:', result.valid)
```

## Using the Python SDK

```python
from uts import UTSClient
import hashlib

# Initialize client
client = UTSClient(calendar_url='https://lgm1.calendar.test.timestamps.now')

# Create digest from file
with open('document.txt', 'rb') as f:
    file_hash = hashlib.sha256(f.read()).hexdigest()

# Submit for timestamping
ots_file, commitment = client.stamp(file_hash)

# Save OTS file
with open('document.txt.ots', 'wb') as f:
    f.write(ots_file)

print(f'Timestamped! Commitment: {commitment}')

# Later: Verify timestamp
result = client.verify(ots_file)
print(f'Verified: {result.valid}')
```

## Common Commands

| Command                  | Description                         |
|--------------------------|-------------------------------------|
| `uts stamp <file>`       | Create timestamp for a file         |
| `uts verify <file.ots>`  | Verify a timestamp                  |
| `uts upgrade <file.ots>` | Upgrade pending to full attestation |
| `uts inspect <file.ots>` | Display timestamp information       |
